## The Problem

India runs hundreds of welfare schemes — PM-KISAN, Ayushman Bharat, widow pensions, scholarships — yet crores of eligible citizens never claim them. The information is scattered across dozens of portals, written in bureaucratic English, and there is no simple way for a daily-wage worker or a rural homemaker to answer one question: **"Which schemes am I actually eligible for, and how do I apply?"**

## The Solution

Scheme Sahayak asks 11 simple questions (1 minute, no login, nothing stored) and returns:

1. **A verified list of central schemes the person is eligible for** — decided by a deterministic rules engine, not by an LLM, so results are reproducible and hallucination-free.
2. **AI-personalised guidance for each scheme** — *why you qualify, exact application steps, document tips* — generated by Gemini in the citizen's own language (English / हिंदी / తెలుగు).
3. **A grounded follow-up chat** — ask anything ("Where do I get an income certificate?") and the AI answers only from the matched schemes' verified data.

## Why the AI is central (and safe)

```
Citizen profile ──► Rules Engine (lib/match.js) ──► eligible schemes + reasons
                                    │
                                    ▼
                    Gemini 2.5 Flash (server-side)
                    grounded ONLY in engine output + curated dataset
                                    │
                                    ▼
        Personalised summary · per-scheme steps · multilingual chat
```

- **Eligibility is a compliance decision** → handled by explainable rules over a curated dataset of 21 major central schemes (`lib/schemes.js`), each with structured criteria and official portal links.
- **Explanation & guidance is a language problem** → handled by Gemini, with strict grounding prompts and JSON-schema output. The model never invents schemes or amounts.
- Graceful degradation: if the AI is unavailable, users still get their verified matches.

## Tech Stack

| Layer | Tech |
|---|---|
| Frontend | React 18 + Vite, responsive, 3-language UI |
| Backend | Vercel Serverless Functions (Node) |
| AI | Google Gemini 2.5 Flash (REST, server-side key) |
| Eligibility | Custom rules engine + curated scheme dataset |
| Tests | 9 unit tests over the matching engine (`npm test`) |

## Run Locally

```bash
npm install
cp .env.example .env          # add your GEMINI_API_KEY (free: aistudio.google.com/apikey)
npm test                      # eligibility engine unit tests
npx vercel dev                # runs frontend + /api functions together
```

## Deploy (Vercel)

1. Push this repo to GitHub (public).
2. Import into Vercel → framework auto-detected (Vite).
3. Add environment variable `GEMINI_API_KEY`.
4. Deploy. `/api/*` functions ship automatically.

## Project Structure

```
api/recommend.js    # profile → engine matches → Gemini personalisation (JSON)
api/chat.js         # grounded multilingual follow-up Q&A
lib/schemes.js      # curated dataset: 21 central schemes + structured criteria
lib/match.js        # deterministic eligibility engine (explainable, tested)
lib/gemini.js       # thin Gemini REST wrapper (no SDK, fast cold starts)
src/                # React app: language select → profile form → results → chat
tests/match.test.js # 9 unit tests
```

## Disclaimer

Scheme rules change over time. Scheme Sahayak provides guidance with official portal links — always verify on the official portal before applying. No user data is stored.
